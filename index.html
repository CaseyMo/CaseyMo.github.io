基于Windows操作系统利用Hexo框架+GitHub Pages搭建个人博客

一、基本概念
Hexo是一个快速、简洁且高效的博客框架，使用 Markdown（或其他渲染引擎）解析文章，在几秒内即可利用靓丽的主题生成静态网页。关于Hexo的更多内容详见Hexo的官方网站。
GitHub Pages是一个静态网站托管服务，能够从建立在GitHub上的仓库中直接存取HTML、CSS和JavaScript文件。你可以通过构建过程来运行这些文件和发布网站。关于GitHub Pages的更多内容详见GitHub Pages的官方网站。

三、建立GitHub Pages
要利用GitHub Pages服务来托管我们的博客，首先需要在GitHub网站注册一个账号。注册好账号后我们就能够创建属于自己的GitHub Pages服务了。
首先登录自己的GitHub账户，在自己的主页点击“Repositories”，点击后将会显示自己的所有仓库。然后点击“New”新建一个名为“username.github.io”的仓库。
注意：username是你的用户名，需要为全小写状态。

五、将博客绑定到自己的域名
我们可以将博客绑定到自己专属的域名。如果暂时还没有购买域名，我们可以在提供域名购买服务的网站（如：万网、GoDaddy等）上进行购买。
购买好自己的专属域名后，我们可以利用阿里云控制台或者DNSPod来做一个域名解析服务。在域名解析平台登录成功后，可以添加或选择自己的域名来配置域名解析。点击“添加记录”按钮，创建一个类型为CNAME的域名解析，CNAME类型的解析代表将一个域名解析到另一个域名。
完成域名解析等步骤后，还不能马上通过自己的域名访问到博客项目，最后需要在博客项目的source文件夹内添加一个名为“CNAME”的文件（请注意这个文件的名字是全大写字母的），并将自己的专属域名名称写入该文件中。

六、总结
在进行博客搭建的过程中虽然遇到了一些问题，但最后还是成功了。在这个过程中，总结了以下几点经验：
1、因为自己的GitHub用户名中含有大写英文字符，在建立仓库时仓库的名称中也包含了相应的大写字符，最后通过查找相关资料，需要确保仓库名均为小写状态。
2、每一次在重新尝试建立blog项目时，修改“_config.yml”文件后，都需要安装名为hexo-deployer-git的部署器插件。
3、在首次将Hexo部署到GitHub Pages时，执行完hexo generate -d命令后，需要在终端窗口提示中输入自己的Personal access tokens。
4、最后，最重要的一点就是在搭建博客的过程中一定要有耐心，按照步骤进行，有时可以删掉该blog文件重新再来。搭建过程中遇到问题，找寻相应的解决办法。
